## 0. แนวคิดหลักที่เปลี่ยน

ราคาวัสดุกับราคาผลิตมีธรรมชาติต่างกัน จึงแยกเป็นคนละเอกสาร:

| | ราคาวัสดุ (PM/RM) | ราคาผลิต |
|---|---|---|
| ผูกกับ | **วัสดุ** (ขวดแก้ว 50ml / หัวน้ำหอม X) | **สินค้า + ชั้นจำนวน** |
| ใช้ซ้ำ | ได้หลายสินค้า หลายงาน → เก็บใน**คลังกลาง** | ของงานนั้นงานเดียว |
| ใครให้ราคา | RD (RM ฿/กก.) / PC (PM ฿/ชิ้น) | ผู้บริหาร (คนเดียวจบ + ลายเซ็น) |
| หมดอายุเมื่อ | supplier ขึ้นราคา / เกินอายุราคา | สเปกหรือจำนวนเปลี่ยน → revise |

```
╔═ เอกสาร 1: ใบขอราคาวัสดุ (MR-YYMMXXXX) ═══════════╗
║ เซลถาม → RD ตอบ RM · PC ตอบ PM                      ║
║ ไม่ผูกสินค้า/ดีล ไม่ผ่านผู้บริหาร                     ║
╚═══════════════════╤═════════════════════════════════╝
                    ↓ คำตอบเข้าอัตโนมัติ
        ┌───────────────────────────┐
        │   คลังราคาวัสดุกลาง        │  ← มี rev, มีอายุ, ทับรายลูกค้าได้
        └───────────┬───────────────┘
                    ↓ ดึงมาใช้ (เซลกดเอง — ไม่เติมอัตโนมัติ)
╔═ เอกสาร 2: ใบขอราคาผลิต (CR-YYMMXXXX) ════════════╗
║ กางบรรทัดตามแม่แบบประเภทสินค้า → เติมราคาจากคลัง     ║
║ ครบทุกบรรทัดบังคับ = ปลดล็อกส่งผู้บริหาร             ║
║ ผู้บริหารเคาะราคาผลิตต่อชั้นจำนวน รายสินค้า + ลายเซ็น ║
║ → ผูก FG (ถ้าไปต่อ) → ป้อนราคาผลิตเข้า FG           ║
╚═════════════════════════════════════════════════════╝
```

ใบขอราคาวัสดุกินหน้าที่ "ขอราคาเก็บข้อมูล" ไปด้วย — เซลถามราคาขวดไว้ดูก่อนได้เสมอ
คำตอบเข้าคลังให้ทุกคนค้นเจอ ไม่ต้องมีชนิดใบพิเศษอีกชั้น

## 1. มติที่ล็อกแล้ว (2026-07-23)

1. **แยก 2 เอกสาร** ตามตารางข้างบน — ใบขอราคาผลิตดึงราคาวัสดุจากคลัง ไม่ถาม RD/PC ตรงอีก
2. **คลังราคาวัสดุมี rev**: PC/RD แก้ราคา = ออก rev ใหม่ ของเก่าเก็บเป็นประวัติ
   ใบที่เคยอ้าง rev เก่ายังชี้ rev เดิม (ตัวเลขไม่เพี้ยนย้อนหลัง)
3. **ราคาเกินอายุ → "ขอยืนยันราคา" ในใบขอราคาผลิตนั้นเลย ไม่ออกเอกสารใหม่**
   PC ยืนยัน/แก้ = ออก rev ใหม่ในคลัง งานถัดไปได้ราคาสดด้วย (มติ: "ออกเป็น revise")
4. **ไม่เติมราคาอัตโนมัติ** — ระบบโชว์ราคากลาง + วันที่อัปเดตข้างช่อง เซลกด "ใช้ราคานี้" เอง
5. **ราคาทับรายลูกค้า** — ลูกค้าเงื่อนไขพิเศษมีราคาเฉพาะได้ ระบบโชว์ตัวทับก่อนราคากลาง
6. **เลิกบังคับผูกดีล** — ดีลและ FG เป็น optional ทั้งคู่ (เหตุผล: "บางสินค้าที่ขอราคาผลิต
   ก็อาจไม่ได้ไปต่อ") ชื่อสินค้า + ประเภทยังบังคับ
7. **FG เกิดเฉพาะของที่ไปต่อ**: รายการในใบ = ตัวแทนสินค้า (ชื่อ/ประเภท/กลิ่น/สูตร/รูปแนบ)
   พอเพียงให้ผู้บริหารเคาะราคา; "ไปต่อ" = เซลกด **"ขึ้นทะเบียน FG จากรายการนี้"**
   (ฟอร์มสินค้า prefill จากใบ) หรือ "ผูก FG เดิม"; "ไม่ไปต่อ" = ไม่ต้องทำอะไร
   ใบเป็นหลักฐานถาวร; ปุ่มป้อนราคาผลิตเข้า FG ล็อกจนกว่าจะผูก FG
8. **Revise ใบขอราคาผลิต = ออกใบใหม่อ้างใบเดิม (rev.2)** — คัดลอกบรรทัด/ราคา
   แก้เฉพาะที่เปลี่ยน ขออนุมัติใหม่ ใบเดิมคงเป็นหลักฐาน เทียบ rev ต่อ rev ได้
9. **RM ผูกสูตร** — snapshot `formulaName/formulaCode/formulaDate` (มีบน products แล้ว
   mig 0112) ลงรายการตอนผูก FG/ขอราคา; สูตรบนสินค้าเปลี่ยนทีหลัง → เตือน ไม่บล็อก
10. มติเดิมที่คงอยู่: อนุมัติรายสินค้า + ตัวนับ x/y นับสด · ผู้บริหารคนเดียวจบ ไม่มี
    อนุมัติซ้อน · executive ไม่มี products:margin · แม่แบบ = admin + ซ่อนแทนลบ ·
    คำศัพท์ทั้งระบบ = "ราคาผลิต" (PR #651)

**สมมติฐานที่ยังปรับได้ (ไม่บล็อกการเริ่ม):** อายุราคาวัสดุ default **90 วัน**
ปรับได้ที่หน้าตั้งค่าคลัง — ถ้าผู้ใช้อยากได้ค่าอื่นแจ้งตอน UAT

## 2. โมเดลข้อมูลเพิ่ม/แก้ (migration 0143–0145)

เลขล่าสุดตอนเขียน = 0142 (ป้อนราคาผลิต — ⚠️ เช็คว่ารันบน prod แล้วหรือยัง)
ของเดิมทั้ง 6 ตาราง **ไม่รื้อ** — เพิ่มตารางใหม่ + ผ่อนข้อบังคับ + เพิ่มคอลัมน์

### `0143_material_price_library.sql` — คลังราคาวัสดุ + ใบขอราคาวัสดุ
```
material_prices                  -- "วัสดุ" หนึ่งตัวในคลัง (หัวเรื่อง ไม่มีราคา)
  id, kind ('RM_F'|'RM_FB'|'PM'), label (เช่น 'ขวดแก้ว 50ml ทรงเหลี่ยม'),
  "sourceDept" ('RD'|'PC'), "customerId" null = ราคากลาง / มีค่า = ทับรายลูกค้า,
  "supplierNote", "isHidden" (ซ่อนแทนลบ — แพตเทิร์น 0140), audit cols
material_price_revisions         -- ราคาจริงเป็นรุ่น — ห้ามแก้ ห้ามลบ (guard)
  id, "materialId", "revisionNo" (1,2,…), "pricePerKg"/"pricePerUnit" (ช่องตามหน่วย
  — CHECK แพตเทิร์น 0141), "quotedById/Name", "quotedAt", "validUntil" null = ใช้
  อายุ default, "sourceRequestId" null = PC แก้เอง/ยืนยันจากใบผลิต, note
material_price_requests          -- ใบขอราคาวัสดุ MR-YYMMXXXX (next_entity_number 'MR')
  id, "docNo", status ('draft'|'pending'|'answered'|'cancelled'),
  "requestedById/Name", team, note, timestamps + guard ร่างลบได้/ส่งแล้วลบไม่ได้
material_price_request_items     -- บรรทัดคำถาม → จับคู่/สร้างวัสดุในคลังตอนตอบ
  id, "requestId", kind, label, "sourceDept", "materialId" null จนกว่าจะตอบ,
  "answeredRevisionId" null จนกว่าจะตอบ, "priceStatus" ('pending'|'quoted')
```
- RD/PC ตอบ = สร้าง/หา `material_prices` + insert revision ใหม่ + ผูกกลับบรรทัด
  (ตอบที่เดียว ราคาเข้าคลังเลย — คลังโตเองจากการถาม-ตอบ ไม่มีใครต้องกรอกมือ)
- คิว/สิทธิ์ลอกของเดิม: canQuoteCosting (ฝ่ายตนเท่านั้น) + chat space 'rd'/'pc'

### `0144_costing_components_use_library.sql` — ใบขอราคาผลิตดึงจากคลัง
```
costing_item_components เพิ่ม:
  "materialId", "materialRevisionId"   -- อ้างคลัง (soft ref): ราคามาจากรุ่นไหน
  "priceSource" ('library'|'confirmed'|'manual')
  -- ช่อง pricePerKg/pricePerUnit เดิมยังเป็น snapshot ของตัวเลขที่ใช้จริง
costing_requests เพิ่ม:
  "confirmRequestedAt/By" ระดับบรรทัดอยู่บน components:
  "confirmStatus" null|'pending'|'confirmed'  -- ปุ่ม "ขอยืนยันราคา" รายบรรทัด
```
- เส้น `/quote` เดิม (RD/PC ตอบในใบผลิต) **ปลดระวาง** → เหลือเส้น `/confirm`:
  PC/RD กดยืนยันบรรทัดที่ถูกขอ → สร้าง revision ใหม่ในคลัง + อัปเดต snapshot บรรทัด
- ด่านส่งผู้บริหารเดิมคงอยู่: บรรทัดบังคับครบ + ต้นทุนคำนวณได้ **+ ราคาไม่เกินอายุ**
  (เกินอายุ = ต้องยืนยันก่อน — ไม่ให้ผู้บริหารเคาะบนราคาเก่า)

### `0145_costing_optional_deal_revise_formula.sql`
```
costing_requests:  "dealId" ผ่อนเป็น null ได้ (ใบไม่มีดีล = สำรวจ) +
                   "baseRequestId", "revisionNo" default 1   -- สาย revise
costing_request_items: "formulaName/formulaCode/formulaDate" snapshot (mig 0112)
guard เพิ่ม: ใบที่มี revision ใหม่กว่าแก้ไม่ได้ (ผู้ใช้ต้องไปทำงานบน rev ล่าสุด)
```

## 3. หน้าจอ / การใช้งาน

| ที่ | ของใหม่/เปลี่ยน |
|---|---|
| `/sa/materials` (ใหม่) | **คลังราคาวัสดุ** — ค้นตาม ชนิด/ชื่อ/ลูกค้า เห็นราคาปัจจุบัน + ประวัติ rev + อายุ; RD/PC แก้ราคาตรงได้ (= ออก rev ใหม่) |
| `/sa/materials/requests` (ใหม่) | ใบขอราคาวัสดุ — เซลเปิดใบถาม, RD/PC ตอบ (คิวแบบเดิม), ตอบแล้วเข้าคลัง |
| `/sa/costing/[id]` (แก้) | บรรทัดวัสดุ: ช่องราคา + ข้าง ๆ โชว์ "ราคากลาง 12.50฿ · อัปเดต 15 พ.ค. ⚠️เกินอายุ" + ปุ่ม **ใช้ราคานี้** / **ขอยืนยันราคา** / ลิงก์เปิดใบขอราคาวัสดุ (ถ้าคลังไม่มี) |
| `/sa/costing/[id]` (เพิ่ม) | รายการอนุมัติแล้ว: ปุ่ม **"ขึ้นทะเบียน FG จากรายการนี้"** (ฟอร์มสินค้า prefill ชื่อ/ประเภท/กลิ่น/สูตร/ลูกค้า) + **"ผูก FG เดิม"** → ปลดล็อกป้อนราคาผลิต |
| `/sa/costing/[id]` (เพิ่ม) | ปุ่ม **"ออกฉบับแก้ไข (rev.2)"** บนใบที่อนุมัติ/จบแล้ว — สร้างใบใหม่อ้างใบเดิม คัดลอกทั้งชุด |
| ฟอร์มเปิดใบ (แก้) | ดีลเป็น optional (เลือกได้แต่ไม่บังคับ — ไม่เลือก = ลูกค้า/ทีมกรอกเอง หรือว่างไว้) |

การแจ้งเตือน: ใบขอราคาวัสดุ → space rd/pc · ขอยืนยันราคา → space ฝ่ายเจ้าของบรรทัด ·
ส่งผู้บริหาร/ผลอนุมัติ → เดิม (executive/sales)

## 6. ความเสี่ยง/คำถามค้าง

1. อายุราคา default 90 วัน — ยืนยันตอน UAT
2. วัสดุชื่อซ้ำ/สะกดต่างกันในคลัง (ขวดแก้ว 50ml vs ขวด 50 ml.) — เฟสแรกพึ่งการค้น +
   ให้ RD/PC เป็นคนจับคู่ตอนตอบ; ถ้าเริ่มรกค่อยทำตัวช่วย merge
3. mig 0142 (ป้อนราคาผลิต) — **เช็คว่ารันบน prod แล้วหรือยัง** ก่อนเริ่ม PR-A
