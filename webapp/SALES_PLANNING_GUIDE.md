# คู่มือใช้งานระบบ "แผนงานขาย" (Sales Planning)

ศูนย์กลาง**งานขายก่อนปิดการขาย** — รับลีด → ตั้งเป้า → ลงดีล → พยากรณ์ยอด → ใบเสนอราคา → ปิดการขาย
แล้วส่งต่องานให้ PM / คลัง / ภาษี ต่อเนื่องบนข้อมูลชุดเดียวกัน

> เอกสารสถาปัตยกรรม/ขอบเขต ดู [`SALES_PM_ROADMAP.md`](SALES_PM_ROADMAP.md) · ไฟล์นี้เป็น **คู่มือผู้ใช้**

---

## แนวคิดหลัก (เข้าใจก่อนใช้)

```
รับลีด → เปิดดีล → วาง forecast → ผูกโครงการ/PM → ทำใบเสนอราคา → Won พร้อมหลักฐาน
                                      ↓                         ↓
                                  ไทม์ไลน์                 คลัง / ภาษี
```

**กฎทองที่ต้องจำ**
- **"Won" ทำจากใบเสนอราคาเท่านั้น** และต้องมีหลักฐานอย่างน้อย 1 ไฟล์ พร้อมประเภท/วันที่เอกสาร
- ตั้งเป้าได้เฉพาะ **หัวหน้าฝ่ายขาย (`ae_supervisor`) / admin**
- **AE เห็นและแก้ได้เฉพาะดีลของตัวเอง**; AC / Senior AE เห็นและแก้ได้ทั้งทีม
- **ราคาในใบเสนอราคาถูกแช่แข็ง** ณ วันที่สร้าง (แก้ราคาสินค้าทีหลังไม่กระทบใบเก่า)

**เมนูหลักตามสิทธิ์**: แดชบอร์ด · ลีด · ดีล · โครงการ · ใบเสนอราคา · งานของฉัน และเมนู **วางเป้า** สำหรับหัวหน้าฝ่ายขาย/admin

**สถานะดีล (stage)**: ลีด → ผ่านคัดกรอง → เสนอราคา → เสนอไทม์ไลน์ → รอยืนยัน → รอมัดจำ → ปิดได้ (Won) → เข้าโครงการ / ไม่สำเร็จ (Lost)

---

## 1. ตั้งเป้าหมาย (สำหรับหัวหน้าฝ่ายขาย/admin) — เมนู "วางเป้า"

**ใครทำ:** หัวหน้าฝ่ายขาย (`ae_supervisor`) และ admin (ทุกทีม)

1. เข้า **บริหารงานขาย → วางเป้า**
2. เลือก **เดือน** (มุมขวาบน)
3. กด **เพิ่ม target**
   - **เดือนเป้าหมาย**: เดือนที่จะวัดผล
   - **ทีม**: เลือกทีมที่ต้องการวางเป้า
   - **เจ้าของเป้า**:
     - **"ทั้งทีม (เป้าทีม)"** = เป้ารวมของทีม
     - **ชื่อ SA** = เป้ารายบุคคล (รายชื่อดึงจากผู้ใช้จริงในระบบ)
   - **ยอดเป้าหมาย**: จำนวนเงิน
4. ด้านล่างมีชิป **"ยังไม่ได้ตั้งเป้ารายบุคคลเดือนนี้"** — คนในทีมที่ยังไม่มีเป้า กดชื่อเพื่อตั้งเป้าให้เลย (สะดวกตอนมีคนเข้าใหม่/ออก)

> ⚠️ ตั้ง **แบบใดแบบหนึ่ง**: เป้าทีม *หรือ* เป้ารายคน (ไม่ตั้งซ้อนในทีมเดียว ยอดจะซ้ำ)

---

## 2. ลงดีลและวาง Forecast (สำหรับ AE) — เมนู "ดีล (Pipeline)"

**ใครทำ:** AE / AC / Senior

1. เข้า **แผนงานขาย → ดีล (Pipeline)**
2. กด **เพิ่ม deal** กรอก:
   - **ชื่อดีล** + **ลูกค้า** (ผูกจากฐานข้อมูลลูกค้า หรือปล่อยว่างช่วงเป็นลีด)
   - **สถานะ**: ไล่ตามความคืบหน้า
   - **เดือนพยากรณ์**: เดือนที่คาดว่าจะปิดได้ (= เดือนที่จะนับ forecast)
   - **มูลค่าโครงการ** = ยอด forecast เต็มของดีล ส่วนโอกาส (%) ใช้แสดงความมั่นใจและสถานะ ไม่ได้นำมาถ่วงยอด forecast
   - **คาดปิดได้ (วันที่)**, **ได้รับมัดจำแล้ว** (ติ๊กเมื่อได้มัดจำจริง)
3. ค้นหา/กรองตามสถานะได้จากแถบด้านบน

**คอลัมน์ในตารางดีล = ปุ่มควบคุมทั้งหมดของดีลนั้น** (ดูข้อ 3–6, 8)

---

## 3. ทำใบเสนอราคา — ปุ่ม "ใบเสนอ"

1. เปิดดีลหรือเมนู **ใบเสนอราคา** → กด **สร้างใบเสนอราคา**
   - ⚠️ ต้อง **ส่งต่อ PM ก่อน** (ให้มีโครงการ + FG) ระบบจึงดึงรายการ FG + จำนวนมาทำใบเสนออัตโนมัติ
   - **ราคาถูกแช่แข็ง** ณ วันที่สร้าง
2. หนึ่งดีลสร้างใบเสนอราคาได้หลายใบตราบใดที่ดีลยังไม่ Won; ใบเดิมแก้แบบ revision เพื่อเก็บประวัติได้
3. ระบบยกเลิกขั้นอนุมัติใบเสนอราคาแล้ว
4. เมื่อลูกค้าตกลง กด **Won** ที่ใบเสนอราคา แล้วระบุหลักฐาน:
   - ประเภท: สลิปชำระเงิน / PO / หนังสือยืนยันคำสั่งซื้อ
   - วันที่เอกสาร และไฟล์แนบอย่างน้อย 1 ไฟล์
   - PO/หนังสือยืนยันคำสั่งซื้อต้องระบุกำหนดชำระ
5. ระบบนับยอดจริงจากยอดใบเสนอราคาก่อน VAT และปิดใบเสนอราคาอื่นในดีลเป็น `closed`

---

## 4. ส่งต่อ PM (ทำไทม์ไลน์) — ปุ่ม "สร้าง PM"

1. ที่แถวดีล กด **สร้าง PM**
2. ระบบสร้างโครงการใน PM + ผูกกับดีล (คอลัมน์เปลี่ยนเป็นลิงก์ **PM**)
3. ไปทำไทม์ไลน์/งานต่อที่ระบบ PM ได้เลย (โครงการเกิดก่อนปิดการขายได้ = ใช้โชว์ไทม์ไลน์ให้ลูกค้า)

---

## 5. ปิดการขาย — ที่ใบเสนอราคา

1. เปิดใบเสนอราคาที่ลูกค้ายืนยัน แล้วกด **Won** (เขียว)
2. แนบหลักฐานตามข้อ 3; ดีลต้องผูกโครงการแล้ว
3. ระบบตั้งดีลเป็น **ปิดได้ (Won)** + นับยอดจริงตามเดือนของวันที่เอกสาร + ปิด forecast ของดีลนั้น
4. ดีลที่ปิดแล้วหายจาก forecast ที่เหลือ ไปโผล่ในช่อง "ปิดได้" บนภาพรวม

---

## 6. ออกเอกสารส่งของ

ฟีเจอร์ปุ่มส่งของในหน้า Sales ถูกปิดไว้ชั่วคราว (`SALES_FEATURES.shipment = false`) งานส่งต่อให้จัดการจากโครงการ/PM จนกว่าจะเปิดฟีเจอร์นี้อีกครั้ง

---

## 7. ติดตามผล (สำหรับหัวหน้า/ผู้บริหาร) — เมนู "ภาพรวม"

1. เข้า **แผนงานขาย → ภาพรวม** เลือกเดือน
2. **การ์ด KPI**: เป้า / คาดการณ์ / มูลค่าดีลเปิด / ปิดได้
3. **ตาราง "เป้า vs ยอดจริง"** — แยก **รายคน (SA)** และ **รายทีม**: เป้า · ปิดได้ · คาดการณ์ · ส่วนต่าง · % (coverage)
4. แผง **ทบทวนพยากรณ์ยอด** และ **ตรวจย้อน FC สหมิตร** มี backend รองรับ แต่ UI ถูกปิดไว้ชั่วคราวด้วย feature flag

---

## 8. ดูข้อมูลลึกของดีล — ปุ่ม "360"

กด **360** ที่แถวดีล → หน้ารวมทุกอย่างของดีล (อ่านอย่างเดียว): สถานะ, มูลค่า, ใบเสนอที่รับ, งานผลิต (PM),
งานปลายน้ำ (ทะเบียนภาษี / PO สหมิตร), ประวัติการเปลี่ยนสถานะ

---

## เส้นทางพิเศษ: ทีม KA (สหมิตร)

ทีม KA ไม่ต้องทำใบเสนอราคา — เมื่อ **ได้รับ PO** ในระบบสหมิตร กดสร้างโครงการ ระบบจะเด้งสร้างโครงการ
**RE-ORDER** + นับเป็นดีลที่ปิดได้อัตโนมัติ (ข้าม pipeline)

---

## สรุปสิทธิ์ตามบทบาท

| บทบาท | ดูดีล | แก้ดีล | ตั้งเป้า | ทบทวน forecast |
|---|---|---|---|---|
| AE | เฉพาะของตัวเอง | เฉพาะของตัวเอง | ✗ | ✗ |
| AC | ทั้งทีม | ทั้งทีม | ✗ | ✗ |
| Senior AE | ทั้งทีม | ทั้งทีม | ✗ | ✗ |
| หัวหน้าฝ่ายขาย / admin | ทุกทีม | ทุกทีม | ทุกทีม | ✓ |

หมายเหตุ: ระบบยกเลิกการอนุมัติใบเสนอราคาแล้ว; คอลัมน์สุดท้ายหมายถึงการทบทวน forecast ซึ่ง UI ยังปิดอยู่

---

## คำถามพบบ่อย

- **ทำไมกด "สร้างใบเสนอราคา" ไม่ได้?** ต้องกด **สร้าง PM** ก่อน แล้วเพิ่ม FG ในโครงการ
- **ทำไมกด Won ที่ใบเสนอไม่ได้?** ดีลต้องผูกโครงการ มียอด/รายการสินค้า และต้องกรอกหลักฐานการสั่งซื้อหรือชำระเงินให้ครบ
- **ทำไมไม่เห็นปุ่มส่ง?** ฟีเจอร์ส่งของจากหน้า Sales ยังปิดอยู่ ให้จัดการผ่านโครงการ/PM
- **ทำไมแก้ดีลของเพื่อนไม่ได้?** AE แก้ได้เฉพาะดีลตัวเอง (ใช้ปุ่ม 360 เพื่อดู)
- **Won แล้วยอดไปไหน?** นับเข้าช่อง "ปิดได้" บนภาพรวม + ตาราง เป้า vs ยอดจริง
